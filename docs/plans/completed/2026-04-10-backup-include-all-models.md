# Fix Prod Backup to Include All Models

## Context

The `backport_data_backup.py` command uses an `INCLUDE_MODELS` allowlist. Any model not explicitly listed is silently excluded from backups. The `accounting` app models (Invoice, Bill, CreditNote, and their line items) were never added, so prod→dev restores have no invoice data. This means:

1. Invoice→Job links are missing in dev, making WIP invoice deductions always zero
2. Any other models added since the list was written are also missing

## Fix

Switch from `INCLUDE_MODELS` (allowlist) to `EXCLUDE_MODELS` (denylist) in `backport_data_backup.py`.

### File: `apps/workflow/management/commands/backport_data_backup.py`

**Step 1:** Replace `INCLUDE_MODELS` with `EXCLUDE_MODELS` — models we deliberately don't want in backups:

```python
EXCLUDE_MODELS = [
    # Framework tables regenerated by migrate
    "contenttypes.contenttype",
    "auth.permission",
    "sessions.session",
    "admin.logentry",
    # XeroPayItem seeded by migration 0187 with fixed UUIDs
    "workflow.xeropayitem",
]
```

**Step 2:** Change the `dumpdata` call to use `--exclude` flags instead of positional model args:

```python
cmd = ["python", "manage.py", "dumpdata"]
for model in EXCLUDE_MODELS:
    cmd += ["--exclude", model]
```

**Step 3:** Update the PII_CONFIG to cover any newly-included models that contain PII (check Invoice, Bill, CreditNote fields — they have `client` FKs but client names are already anonymized via the client model).

**Step 4:** Update the `analyze_fields` MODELS_TO_ANALYZE list similarly, or just remove the filter and analyze everything.
